## Objective

The goal was to identify the most effective AI model and effort level for a student focused on coding, debugging, learning new concepts, and project development. Using Claude's personalized workflow recommendations, I analyzed how different models perform across various technical tasks.

## Strategy

Based on the generated workflow, I adopted the following approach:

### Primary Model: Claude Sonnet

Claude Sonnet was recommended as the default model for most day-to-day coding activities because it provides a strong balance between speed, reasoning capability, and explanation quality.

**Used for:**

* Debugging code
* Learning programming concepts
* Reviewing assignments
* Writing functions and scripts
* Project development
* Interview preparation

### Secondary Models

#### Claude Haiku

Used for lightweight tasks requiring quick responses:

* Syntax checks
* Simple definitions
* Quick programming questions
* Vocabulary lookups

#### Claude Opus

Reserved for highly complex tasks:

* Advanced algorithms
* System design
* Research-level analysis
* Multi-step architectural decisions
* Capstone and final-year projects

---

## Effort-Level Strategy

| Effort Level | Use Case                                                       |
| ------------ | -------------------------------------------------------------- |
| Low          | Quick lookups, syntax checks, simple questions                 |
| Standard     | Daily coding, debugging, concept explanations                  |
| High         | Project design, optimization, interview preparation            |
| Max          | Research work, complex architectures, advanced problem-solving |

### Recommended Workflow

* **Sonnet + Standard Effort** → Default choice for 80–90% of coding tasks.
* **Sonnet + High Effort** → When deeper reasoning or planning is needed.
* **Opus + High/Max Effort** → For highly complex technical challenges.

## Key Learnings

### 1. Model Selection Matters

Different AI models excel at different tasks. Using the most powerful model for every task is often unnecessary and less efficient.

### 2. Sonnet Provides the Best Balance

For a student and developer, Sonnet offers the optimal combination of speed, reasoning, and explanation quality.

### 3. Effort Levels Should Match Task Complexity

Increasing effort levels only when needed improves efficiency and reduces response time.

### 4. Context Improves Results

Providing clear requirements, expected outputs, and project context significantly improves the quality of AI-generated responses.

### 5. AI Works Best as a Learning Partner

Rather than accepting answers directly, asking for explanations, reasoning, and alternative solutions leads to better understanding and skill development.


## Final Takeaway

The most effective workflow for daily coding and learning is:

**Claude Sonnet + Standard Effort**

This combination delivers high-quality explanations, efficient debugging support, and strong coding assistance while maintaining fast response times. More advanced models and higher effort levels should be reserved for genuinely complex problems where deeper reasoning is required.
